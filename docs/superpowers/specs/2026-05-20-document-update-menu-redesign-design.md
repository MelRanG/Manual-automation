# 문서 현행화 메뉴 재설계

## 배경

현재 문서 현행화 프로세스가 여러 메뉴(서비스 요청, 오류 제보, 매뉴얼 생성, 승인 관리, 변경 영향)에 분산되어 있어 흐름이 불명확하다. 승인 관리의 Jira SR 탭과 변경 영향 페이지가 기능적으로 겹친다. 이를 3개 메뉴로 통합해 각 변경 경로가 한 페이지 안에서 완결되도록 재설계한다.

## 문서 변경 경로

1. **매뉴얼 생성** — URL을 입력해 AI가 매뉴얼을 신규 생성하거나 문서 변경 제안을 생성
2. **오류 제보** — 챗봇 대화 중 사용자가 문서 오류를 제보 → AI 수정 초안 생성
3. **Jira SR** — 서비스 요청이 Jira 이슈로 연결되어 상태 추적 → 완료 후 문서 반영

## 네비게이션 변경

### 제거
- 승인 관리 (`/approvals`)
- 변경 영향 (`/change-impact`)

### 섹션명 변경
- "변경 관리" → "문서 현행화"

### 새 메뉴 구성
```
문서 현행화
  ├── 매뉴얼 생성  (/manuals)
  ├── 오류 제보    (/feedback)
  └── Jira SR      (/sr)
```

## 매뉴얼 생성 (`/manuals`)

### 탭
| 탭 | 설명 |
|---|---|
| 전체 | 모든 요청 |
| 검토요청 | AI 초안 생성 완료, 검토 대기 중 |
| 완료 | 승인 및 문서 반영 완료 |

### 목록 항목
- 요청자, 대상 URL, 상태 배지, 요청 일시

### 항목 클릭 → 상세 패널
1. **요청 정보** — 요청자, 일시, 대상 URL, 시나리오 스텝(있을 경우)
2. **AI 생성 초안** — 확인·수정·승인
3. **변경 이력 타임라인**

## 오류 제보 (`/feedback`)

### 탭
| 탭 | 설명 |
|---|---|
| 전체 | 모든 제보 |
| 검토요청 | AI 수정 초안 생성 완료, 검토 대기 중 |

3. **AI 수정 초안** — 확인·수정·승인
4. **변경 이력 타임라인**

## Jira SR (`/sr`)

### 탭
| 탭 | 설명 |
|---|---|
| 전체 | 모든 SR |
| SR요청 대기 | Jira 이슈 생성 전 대기 상태 |
| SR 진행중 | Jira 이슈 진행 중 |
| 검토 | Jira 완료 후 문서 반영 검토 단계 |
| 완료 | 문서 반영 완료 |

### 필터
- 출처 태그: `직접생성` / `챗봇`

### 목록 항목
- SR 제목, 요청자, 출처 태그, Jira 이슈 번호, 상태 배지, 요청 일시

### 항목 클릭 → 상세 패널

**요청 정보 영역**
- 요청자, 일시, 출처(직접생성/챗봇), Jira 이슈 링크, SR 설명

**검토 단계 (Jira 완료 후 활성화)**

Step 1. 문서 반영 방식 선택
- 신규 문서 작성
- 기존 문서 수정

Step 2. 기존 문서 수정 선택 시 — 반영할 문서 선택
- 문서 검색·선택 UI

Step 3. AI 초안 확인
- diff 비교 뷰 (기존 내용 vs AI 수정안)
- 직접 수정 가능
- 승인 → 문서 반영

**변경 이력 타임라인**

## 공통: 변경 이력 타임라인

모든 페이지의 항목 상세 패널에 포함. 완료 후에도 목록에 남아 이력 조회 가능.

### 표시 형식
```
[생성]    홍길동 · 2026-05-20 10:00   "페이지명 업데이트" 요청
[AI 초안] 자동 생성 완료
[수정]    김철수 · 2026-05-20 14:30   초안 일부 수정
[승인]    박영희 · 2026-05-20 15:00   승인 및 문서 반영
```

### 이력에 포함되는 이벤트
- 요청 생성 (요청자, 출처)
- AI 초안 생성
- 수정 (수정자, 수정 내용 요약)
- 상태 변경 (변경자, 변경 전→후)
- 승인 (승인자)
- 문서 반영

## 데이터 모델 변경 사항

### 기존 유지
- `sr_drafts` 테이블 — status 필드 값 매핑 변경
- `feedback_reports` 테이블 — status 필드 값 매핑 변경
- `manual_jobs` 테이블

### status 매핑 (SR)
| 기존 status | 새 탭 |
|---|---|
| `draft` | SR요청 대기 |
| `active` | SR 진행중 |
| `pending_doc_review` | 검토 |
| `done` | 완료 |

### status 매핑 (오류 제보 — feedback_reports)
| 기존 status | 새 탭 |
|---|---|
| `pending` | 검토요청 |
| `processed` | 완료 |

### status 매핑 (매뉴얼 생성 — proposed_document_changes.status, source_type='manual')
| 기존 status | 새 탭 |
|---|---|
| `pending` | 검토요청 |
| `approved` / `applied` | 완료 |

### 이력 테이블 신규 추가 고려
- 각 항목의 이벤트 로그를 저장하는 `change_history` 테이블
- `entity_type` (sr / feedback / manual), `entity_id`, `event_type`, `actor_id`, `detail`, `created_at`

## 구현 범위

### Phase 1 — 네비게이션·라우팅
- Layout.tsx 네비게이션 섹션명·메뉴 변경
- 승인 관리, 변경 영향 라우트 제거

### Phase 2 — 탭 재구성
- 매뉴얼 생성: 탭 전체/검토요청/완료
- 오류 제보: 탭 전체/검토요청/완료
- Jira SR: 탭 전체/SR요청대기/SR진행중/검토/완료 + 출처 태그 필터

### Phase 3 — 상세 패널 (항목 클릭)
- 각 페이지 항목 클릭 시 슬라이드 패널 또는 모달
- SR 검토 단계 Step 1~3 흐름

### Phase 4 — 변경 이력 타임라인
- `change_history` 테이블 및 API
- 상세 패널 하단 타임라인 UI
