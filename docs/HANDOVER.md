# Manual Automation — 인수인계 문서

> 개발 현황, 구현된 기능, 미흡한 부분, 우선순위 정리.

---

## 서비스 한 줄 설명

사내 문서를 등록하면 AI가 질의응답하고, 오류 제보 → AI 수정안 생성 → 담당자 승인 → 문서 자동 업데이트까지 이어지는 **문서 생명주기 자동화 플랫폼**.

---

## 현재 구현된 기능 (사용자 기준)

### 1. 로그인
- 이메일 입력만으로 로그인 (비밀번호 없음)
- 없는 이메일이면 자동으로 계정 생성
- 빠른 로그인: `admin@docops.ai` 버튼 제공
- 로그인 후 모든 페이지 이용 가능, 미로그인 시 로그인 화면으로 리다이렉트

### 2. 문서 관리 (`/documents`)
- 문서 목록 조회, 신뢰도 점수 / 상태 / 우선순위 표시
- 텍스트 파일 업로드 가능 → 내용이 문서로 등록됨
- 문서 상세 페이지(`/documents/:id`): 내용 열람, 버전 이력 확인
- **⚠️ 미흡**: DOCX/XLSX/PPTX 업로드 시 내용이 깨짐. 웹 뷰어 미구현 (하단 참고)

### 3. Q&A 챗봇 (`/chat`)
- 대화 세션 생성 → 질문 입력 → AI가 등록된 문서 기반으로 답변
- 답변 스트리밍 (SSE), 참고 문서(출처) 인용 표시
- 신뢰도 낮은 문서 참조 시 경고 표시
- 답변에 오류 있으면 "실제와 달라요" 버튼으로 바로 오류 제보
- 대화 내역 날짜별 그룹 표시, 세션 삭제 가능
- **⚠️ 미흡**: AWS Bedrock 키 없으면 Mock 모드로 동작 → 실제 문서 기반 답변 안 됨 (아래 우선순위 #1 참고)

### 4. 오류 제보 (`/feedback`)
- 오류 내용 + 관련 문서 ID 입력 → 제보 접수
- AI가 자동으로 수정안(diff) 생성 → 승인 관리로 이동
- 제보 목록 조회, 삭제 가능
- 제보 접수 시 해당 문서 소유자에게 실시간 알림 발송

### 5. 승인 관리 (`/approvals`)
- 오류 제보로 생성된 AI 수정안을 검토
- 승인 → 문서에 새 버전으로 반영
- 반려 → 피드백 작성자에게 알림
- 편집 후 승인 가능 (수정안 직접 고쳐서 반영)

### 6. 문서 신뢰도 (`/trust`)
- 등록된 모든 문서의 신뢰도 점수 목록
- "재계산" 버튼으로 수동 갱신 가능
- 신뢰도 기준: 최근 검토일, 오류 제보 반영 여부, 버전 업데이트 빈도
- **⚠️ 미흡**: 신뢰도 계산 로직이 단순함. 검토일 기반 자동 하락, 오픈 날짜 등 지표 보강 필요

### 7. 서비스 요청 (`/sr`)
- SR(Service Request) 초안 작성 및 제출
- AI로 SR 자동 생성 (문서 + 이슈 설명 입력 시)
- Jira 웹훅으로 외부 전송 가능 (현재 `.env`에 URL 입력 필요)
- **⚠️ 미흡**: SR 요청과 도메인 요청 구분 기능 구현 요청됨 → 미완

### 8. 변경 영향 분석 (`/change-impact`)
- 특정 문서/SR의 변경이 다른 문서에 미치는 영향 AI 분석
- 추천 전략(즉시 반영 / 단계적 반영 등) 제시

### 9. 매뉴얼 자동 생성 (`/manuals`)
- 웹사이트 URL 입력 → Playwright로 스크린샷 캡처 + 페이지 텍스트 추출 → AI가 한국어 매뉴얼 작성
- 로그인 자격증명 입력 시 자동 로그인 후 캡처
- 시나리오 단계별 캡처 지원
- 생성 완료 시 문서로 자동 등록
- **⚠️ 미흡**: Bedrock/API 키 없으면 Mock 텍스트로 생성됨

### 10. 알림 시스템
- 헤더 우측 벨 아이콘: 미읽음 알림 수 배지
- 클릭 시 드롭다운으로 알림 목록
- 새 알림 수신 시 우하단 토스트 팝업 (4초 자동 닫힘)
- 트리거: 오류 제보 접수, 승인/반려 결정
- "전체 읽음" 처리 가능

### 11. 위젯 (`/widget-demo`)
- 외부 사이트에 임베드 가능한 챗봇 위젯 데모 페이지
- **⚠️ 미흡**: UI 수정 필요 — 아시아나항공/한진택배 등 실제 고객사 화면에 자연스럽게 뜨도록 디자인 개선 필요

### 12. Jira 웹훅 로그 (`/webhook-logs`)
- 외부 Jira로 전송된 웹훅 요청/응답 이력 확인

## 우선순위별 미해결 항목

### 🔴 1순위 — AWS Bedrock 연동 (챗봇 실제 동작)

현재 `LLM_PROVIDER=bedrock`으로 설정되어 있으나 AWS 키가 없어 실제 LLM 호출 불가. Mock 모드는 문서 내용을 무시하고 고정 텍스트를 반환함.

**필요한 것:**
- AWS 계정에서 Bedrock 접근 권한 활성화 (us-east-1 권장)
- IAM 사용자 생성 → Access Key 발급
- `.env` 파일에 입력:
  ```
  AWS_ACCESS_KEY_ID=AKIA...
  AWS_SECRET_ACCESS_KEY=...
  AWS_REGION=us-east-1
  BEDROCK_MODEL_ID=us.anthropic.claude-sonnet-4-6-20251101-v1:0
  ```
- 키 발급 전 임시 방편: Anthropic API 키 또는 OpenAI API 키로도 전환 가능 (`LLM_PROVIDER=anthropic` 또는 `openai`)

연동 후 반드시 실제 문서 기반 Q&A 테스트 필요. 임베딩도 현재 Mock 상태이므로 벡터 검색까지 실제로 동작하려면 `EMBEDDING_MODEL` 설정도 확인 필요.

---

### 🔴 2순위 — Jira 양방향 연동

현재 상태: SR 제출 시 Jira 웹훅으로 단방향 전송만 구현. Jira → 시스템 방향(이슈 가져오기)은 미구현.

**필요한 것:**
- `.env`에 `JIRA_WEBHOOK_URL` 입력 → SR 제출 시 Jira에 이슈 자동 생성
- Jira에서 이슈 상태 변경 시 시스템으로 콜백(웹훅) 수신 엔드포인트 구현 필요
- SR 요청 vs 도메인 요청 구분 기능 구현 (기획 확정 후 개발)

---

### 🟡 3순위 — 문서 신뢰도 지수 고도화

현재 신뢰도 점수는 단순 수치만 저장. 실제 지표 기반 자동 계산 필요.

**보강 필요 지표:**
- 최초 등록일로부터 경과 시간
- 마지막 검토일로부터 경과 시간 (오래될수록 하락)
- 누적 오류 제보 수
- 승인 완료된 수정 반영 횟수

---

### 🟡 4순위 — 위젯 UI 수정

`/widget-demo` 페이지에서 고객사(아시아나항공, 한진택배 등) 실제 화면 위에 챗봇이 자연스럽게 떠있는 형태로 UI 개선 필요.

---

### 🟡 5순위 — DOCX/XLSX/PPTX 웹 뷰어

현재 사무용 파일 업로드 시 내용이 깨져서 저장됨. 웹에서 원본 그대로 보이는 뷰어 미구현.

**검토된 방향:**
- **클라이언트 사이드**: `mammoth.js`(DOCX), `SheetJS`(XLSX) — 뷰어만 가능, 편집 불가
- **편집까지 원하면**: ONLYOFFICE를 별도 Docker 컨테이너로 배포 — 구현 복잡도 높음, MVC 이후 우선순위

실제로 구글 독스, MS Office Online도 자체 포맷으로 변환해서 보여주는 방식이므로, 단기적으로는 "업로드 → 다운로드 후 편집 → 재업로드" 워크플로우를 안내하고 장기 과제로 두는 것을 권장.

---

### 🔵 6순위 — 기타

- 위젯 임베딩 고객사별 customization (로고, 색상 등)
- 대시보드 통계 실시간 갱신
- 사용자 권한 관리 (현재 role 필드는 있으나 권한 체크 미구현)

## 개발 환경 세팅 (신규 입사자)

```bash
# 1. 저장소 클론 후
cd Manual-automation

# 2. DB 실행
docker compose up db -d

# 3. .env 파일 생성 (프로젝트 루트)
cp .env.example .env   # 없으면 아래 내용으로 직접 생성
# LLM_PROVIDER=mock 으로 두면 API 키 없이 동작

# 4. 백엔드
cd backend
uv sync
uv run alembic upgrade head   # DB 마이그레이션
uv run uvicorn app.main:app --reload --port 8000

# 5. 프론트엔드 (새 터미널)
cd frontend
pnpm install
pnpm dev

# 브라우저: http://localhost:5173
# API 문서: http://localhost:8000/docs
# 로그인: admin@docops.ai
```

자세한 내용은 `README.md` 참고.
